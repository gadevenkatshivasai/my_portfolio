Wheel data Recording System

Objective / Problem Statement

The Indian Railways required a robust solution to manage the intricate process of wheel set overhauling, specifically focusing on Intermediate Overhauling (IOH) and Periodic Overhauling (POH). The existing manual processes were prone to inefficiencies, data inaccuracies, and lack of real-time tracking, leading to delays and safety concerns. The objective was to develop a web application that would streamline these processes, ensure comprehensive data collection, enhance decision-making, and improve overall efficiency

What We Did

We developed the Wheel Data Recording System (WDRS), a web-based application tailored to the specific needs of Indian Railways. The system digitalized and automated the entire wheel set overhauling process, from initial assessment to final assembly. The application was designed with a modular approach, ensuring each stage of the overhauling process—Receipt, Pre-inspection, Condemnation, Press-in/out, and Dispatch—was meticulously recorded and monitored. Unique identifiers like TID, AID, DID, and BID were implemented to track each component's history and prevent data duplication.

Key Features

  • Modular Approach: The application is divided into distinct modules (Receipt, Pre-inspection, Condemnation, Press-in/out, Dispatch), each tailored to specific stages of the overhauling process.
  • Comprehensive Data Collection: Detailed information on wheel discs, axles, and bearings is captured, enabling precise tracking and historical record maintenance
  • Digitalization Benefits The application transforms manual record-keeping into a digital process, vastly improving data accessibility, retrieval efficiency, and operational transparency.
  • Report Generation Module: Users can generate insightful reports based on specific parameters (e.g., wheel, axle, bearing), facilitating informed decision-making and performance analysis.

  • Front- End HTML, CSS, JavaScript, BootStrap 5
  • Back-end PHP
  • Database Microsoft SQL Server, SQL Server Management Studio (SSMS)

    • Functionality

  • Data Entry & Management: Seamless input of data for wheel sets, axles, discs, and bearings across multiple overhauling stages.
  • Unique Identifiers: Implementation of TID (Transaction ID), AID (Axle ID), DID (Disc ID), and BID (Bearing ID) to uniquely track each component's history and transactions.
  • Real-Time Monitoring: Live updates on the overhauling process, allowing for timely interventions and adjustments.
  • Report Generation: Automated creation of detailed reports that summarize overhauling activities, trends, and performance metrics.

    • Project Implementation

  • Design & Development: The system was designed using Bootstrap for responsive design, ensuring accessibility across devices. PHP was used for backend logic, and Microsoft SQL Server was employed for managing the database.
  • Modular Architecture: The project was implemented in modules, allowing for isolated development, testing, and deployment of each phase of the overhauling process.
  • Database Management: SQL Server Management Studio (SSMS) was used to configure, monitor, and administer the SQL Server database, ensuring efficient data handling and integrity.

    • Project Challengues & Solution

  • Responsive Design Across Devices: Ensuring a seamless user experience across different devices and screen sizes was challenging. The solution involved leveraging Bootstrap's responsive grid system and media queries, along with rigorous testing on multiple devices.
  • Maintaining Design Integrity: Translating UI/UX design mockups into interactive web interfaces required careful attention to detail. By using CSS for styling and JavaScript for dynamic behavior, we preserved design integrity while ensuring functionality.
  • Duplicate Data Prevention: The potential for duplicate data entries due to recurring wheel set overhauls was mitigated by implementing unique identifiers (TID, AID, DID, BID) for each transaction, ensuring data accuracy and consistency.

    • Deployment Details

  • Rollout Strategy: The deployment was executed in phases, starting with a pilot division to test and refine the system. Following successful validation, the system was progressively implemented across additional divisions.
  • Infrastructure: The application was integrated into the existing IT infrastructure, leveraging Bootstrap for a responsive design compatible with various devices used in the railway workshops.
  • Training and Support: Comprehensive training sessions were conducted for users at each location to ensure effective utilization of the system. Ongoing technical support was provided to address any issues and ensure smooth operation.

    • Maintance & Support

  • Maintenance Schedule: Regular maintenance is performed to ensure the system remains operational and up-to-date. This includes routine checks, updates, and bug fixes.
  • Scalability: The system was designed with scalability in mind, allowing for future expansion and integration of additional features or modules as needed. The modular architecture ensures that new functionalities can be added without significant disruptions
  • Documentation Comprehensive documentation is provided, including user manuals, technical guides, and maintenance procedures, to facilitate smooth operations and support.

    • Conclusion

    The Wheel Data Recording System (WDRS) successfully addressed the challenges faced by Indian Railways in managing wheel set overhauling processes. By digitalizing and automating the workflow, WDRS improved efficiency, accuracy, and transparency in maintenance operations. The project not only optimized resource utilization but also upheld safety standards, setting a new benchmark for railway maintenance systems.
